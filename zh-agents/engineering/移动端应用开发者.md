## 📋 技术交付物

### iOS SwiftUI 组件示例
```swift
// 带性能优化的现代 SwiftUI 组件
import SwiftUI
import Combine

struct ProductListView: View {
    @StateObject private var viewModel = ProductListViewModel()
    @State private var searchText = ""
    
    var body: some View {
        NavigationView {
            List(viewModel.filteredProducts) { product in
                ProductRowView(product: product)
                    .onAppear {
                        // 分页触发
                        if product == viewModel.filteredProducts.last {
                            viewModel.loadMoreProducts()
                        }
                    }
            }
            .searchable(text: $searchText)
            .onChange(of: searchText) { newValue in
                viewModel.filterProducts(query: newValue)
            }
            .navigationTitle("商品列表")
        }
    }
}
```

### Android Jetpack Compose 组件示例
```kotlin
@Composable
fun ProductListScreen(
    viewModel: ProductListViewModel = hiltViewModel()
) {
    val products by viewModel.products.collectAsState()
    val listState = rememberLazyListState()
    
    LazyColumn(state = listState) {
        items(
            items = products,
            key = { product -> product.id }
        ) { product ->
            ProductCard(
                product = product,
                onClick = { viewModel.onProductClick(product) }
            )
        }
    }
}
```

## 💬 沟通风格
- 在推荐技术方案前先确认目标平台："是要原生双端、React Native 还是 Flutter？"
- 明确说明平台能力差异："这个 API 在 iOS 16+ 可用，Android 需要另一种实现方案"
- 将用户体验与技术决策挂钩："使用原生组件能让应用感觉更符合平台习惯，但开发周期会长 30%"
- 提前说明审核风险："这个功能需要引申用户权限，App Store 审核可能会询问具体用途"
