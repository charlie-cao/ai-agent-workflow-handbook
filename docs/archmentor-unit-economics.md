# ArchMentor 单位经济模型

> **版本**: v1.0 | **日期**: 2026-03-25  
> **用途**: 验证商业模式财务可行性，指导定价和增长决策

---

## 核心假设

| 参数 | 数值 | 来源/依据 |
|------|------|---------|
| 定价（月付） | $39/月 | 对标 HelloInterview $39/月 |
| 定价（年付） | $199/年 | 主推，等效 $16.6/月 |
| 年付占比预估 | 40% | B2C SaaS 行业基准约 30-45% |
| 月付 Churn Rate | ~18%/月 | EdTech SaaS 行业基准 |
| 年付续费率 | ~55% | 付年费用户 Churn 更低 |

---

## LTV 计算

### 月付用户 LTV
```
LTV_月付 = MRR / Churn Rate = $39 / 18% ≈ $217
```

### 年付用户 LTV（考虑续费）
```
Year 1: $199
Year 2: $199 × 55% = $109
Year 3: $199 × 30% = $60
LTV_年付 ≈ $368
```

### 加权混合 LTV
```
LTV_混合 = $217 × 60% + $368 × 40% ≈ $278
```

## CAC 估算（各渠道）

| 获客渠道 | CAC 估算 | 依据 |
|---------|---------|------|
| Reddit/Twitter 有机 | $0-5 | 时间成本，前期主要渠道 |
| SEO 内容 | $10-20 | 内容制作分摊，延迟 3-6 月见效 |
| Product Hunt 发布 | $15-30 | 约 5000 访客，转化 2-5% |
| 邀请裂变 | $10-15 | 2周免费成本 + 转化损耗 |
| **目标混合 CAC** | **$20-35（均值 $27）** | 前 90 天以有机为主 |

---

## 核心健康指标

### LTV:CAC 比率
```
LTV:CAC = $278 / $27 ≈ 10.3x
```
- > 3x = 良好
- > 5x = 优秀  
- **10x = 有机增长阶段的理想状态**

### Payback Period（回收获客成本时间）
```
Payback = CAC / (MRR × 毛利率)
         = $27 / ($39 × 80%)
         ≈ 0.87 个月
```
**几乎立即回收**——PLG 模式的核心财务优势。

## 成本结构

### 变动成本（每用户/月）

| 成本项 | 估算 |
|--------|------|
| Claude API（约 20 次对话/用户/月 × $0.01/次） | $0.20 |
| Stripe 手续费（2.9% + $0.30） | $1.43 |
| 服务器分摊（Vercel + Railway） | $0.15 |
| **总变动成本/用户** | **≈$1.78/月** |

**毛利率 = ($39 - $1.78) / $39 ≈ 95.4%**（SaaS 行业领先水平）

### 固定成本（月）

| 成本项 | 月估算 |
|--------|-------|
| 工具 SaaS（PostHog, Linear, Resend 等） | $150 |
| 内容创作/运营时间成本（按 $50/h × 20h） | $1,000 |
| 其他杂费 | $200 |
| **月固定成本** | **≈$1,350** |

---

## 盈亏平衡分析

```
Break-even 用户数 = 月固定成本 / 每用户毛利润
                  = $1,350 / ($39 × 95.4% - 其他变动)
                  ≈ $1,350 / $35.6
                  ≈ 38 用户
```

**仅需 38-46 名付费用户即可覆盖所有运营成本**。极低的财务风险。

## 敏感性分析

### 如果年付比例更高（60%）
```
混合 LTV = $217 × 40% + $368 × 60% ≈ $307
LTV:CAC = $307 / $27 ≈ 11.4x (更优)
```

### 如果 Churn 下降至 12%/月（更高留存）
```
LTV_月付 = $39 / 12% = $325
混合 LTV ≈ $342
LTV:CAC ≈ 12.7x
```

**结论**: 核心杠杆点是**留存率**，而留存率取决于用户每次对话后感受到的价值增长。这反过来强化了"苏格拉底式 AI 辅导"的产品设计优先级。
