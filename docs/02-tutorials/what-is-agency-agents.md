# Agency Agents 完整使用教程

> 本文从「是什么」到「怎么用」，全面介绍 agency-agents 工具链。

---

## Agency Agents 是什么

agency-agents 是一个**开源的 AI 专家角色库**，每个「Agent」是一个 `.md` 文件，包含：

```
---
name: 前端开发工程师
description: 专注现代 Web 技术...
---

# 角色定义
你是一个专注于...

## 核心工作流
1. 理解需求
2. ...

## 交付物格式
...
```

当你在 GitHub Copilot Chat 的 **Agent 模式**中选择某个 Agent 时，这个文件的内容会作为系统提示词注入，让 AI 扮演这个特定角色。

---

## 和直接问 AI 有什么区别

| 方式 | 体验 |
|------|------|
| 直接问 Copilot Chat | 通用 AI，什么都懂但什么都不专 |
| 选择「安全工程师」Agent | AI 进入安全专家模式：威胁建模视角、按照 OWASP 标准输出、自动输出漏洞报告格式 |
| 选择「产品经理」Agent | AI 进入 PM 模式：输出 PRD 格式、关注用户故事和验收标准、会自动追问目标用户 |

**比喻**：不是换了一个 AI，而是给同一个 AI 换了不同的「工作手册」。

---

## 三种使用模式

### 模式1：单专家（最简单）
```
一个 Copilot Chat 对话 + 选一个 Agent + 完成一个具体任务
```
适合：写一段代码、审查一个文件、起草一篇文档

### 模式2：流水线（中等复杂度）
```
Agent A 输出 → 你复制 → Agent B 输入 → Agent B 输出 → 你复制 → Agent C...
```
适合：功能开发全流程（设计 → 实现 → 审查 → 文档）

### 模式3：并行（最高效）
```
同时开多个 Copilot Chat 对话，各跑不同 Agent，最后汇总
```
适合：产品立项（市场研究 + 技术架构 + 增长方案同步进行）

---

## 激活 Agent 的方法

1. `Ctrl+Alt+I` 打开 Copilot Chat
2. 点击聊天框左上角的模式/Agent 下拉
3. 选择需要的 Agent（已汉化，如「安全工程师」「产品经理」）
4. 输入你的任务描述

## Agent 分类速查

共 144 个 Agent，分 13 个部门，详见 [安装指南](../01-setup/agency-agents-guide.md#分工目录agent-分类)。

**最常用的 10 个：**

| Agent | 激活时机 |
|-------|----------|
| 软件架构师 | 设计任何新功能/系统时，先让它出方案 |
| 代码审查工程师 | PR 合并前，让它过一遍 |
| 安全工程师 | 涉及认证/支付/敏感数据时，必须过它 |
| 产品经理 | 开始做任何功能前，先定好 PRD |
| Sprint 优先级规划师 | 每次 Sprint 规划时 |
| 技术文档工程师 | 功能完成后写文档 |
| 生产就绪验证员 | 上线前的最终检查 |
| 市场趋势研究员 | 立项前的市场验证 |
| 增长黑客 | 产品发布准备 |
| 多智能体编排师 | 需要协调多个 Agent 完成复杂任务时 |
